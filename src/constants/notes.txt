  Note:
     - Uses Formik for form handling
     - Uses React state for multi-step flow
     - Prevents data overwrite
     - Clean separation of concerns
     - Scalable for more steps
     - Production-ready pattern

     Formik -  manages form state (values, errors, touched, submit)
     Yup    - defines rules for validation (what is valid / invalid)

      Why Yup is used with Formik

     Without Yup, you’d write manual validation logic.
     With Yup, you define validation in a clean, declarative schema.
     - Less code
     - Readable
     - Reusable
     - Handles complex validations easily
